Two-Column

Code

impedit, explicabo, sit repellat tenetur soluta eius quae ducimus vero minima
perspiciatis at illo itaque modi quos delectus nesciunt est facere. Lorem ipsum
dolor sit amet, consectetur adipisicing elit. Quas, excepturi eius hic officiis
at illo natus deserunt necessitatibus dignissimos quo assumenda est odit
voluptatem laudantium dolores alias esse!</p>

Buttons

Buttons are make with a or button element with the btn class. For variations, use info, success, warning, or danger classes.

Regular

Code

<a class="btn default" href="javascript:void(0)">Default</a>
<a class="btn info" href="javascript:void(0)">Info<a>
<a class="btn success" href="javascript:void(0)">Success</a>
<a class="btn warning" href="javascript:void(0)">Warning</a>
<a class="btn danger" href="javascript:void(0)">Danger</a>

Result

Default Info Success Warning Danger

Block-Level

Make the button into a block-level button by adding the block class. For variations, use info, success, warning, and danger classes.

Code

<a class="btn default block" href="javascript:void(0)">Default</a>
<a class="btn info block" href="javascript:void(0)">Info</a>
<a class="btn success block" href="javascript:void(0)">Success</a>
<a class="btn warning block" href="javascript:void(0)">Warning</a>
<button class="btn danger block" href="javascript:void(0)">Danger</button>

Result

Default Info Success Warning

Footer

Footer is always the last layout at the bottom of the page. The texts in the footer are optional, and it is added by the CSS. By default, the texts will be displayed. To remove the texts, add the no-text class to the <footer> element.

Do not remove the <p> element. The <p> element must stay even if it is empty.

Default

Code

<footer>
	<p></p>
</footer>

Result

without CSS text

Code

<footer class="no-text">
	<p></p>
</footer>
